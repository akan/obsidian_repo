文字凝练稿

00:00:02

**曲凯:**

今天请到了invoko.ai的创始人梦琪，梦琪先简单介绍一下自己吧。

**梦琪:**

我大概是去年开始创业，再过一两周就一周年了。创业之前做过蛮多奇怪的事，先做了几年To B，在SaaS比较火的时候做解决方案，最近几年做了一些To C的事，在TikTok，后面甚至还做过一年HR。因为很早就知道自己可能要创业，所以一直抱着能多学点就多学点、学完了自己好用的心态。

00:01:10

**曲凯:**

AI世界的时间让人觉得过得很快，我都觉得你创业很久了，但其实才一年。不过你这一年走完了很多AI软件创业者的典型流程。我总结一下，尤其是去年和前年那一波字节背景的创业者，可能高P出来，组了很好的团队，讲了一个当下AI流行的故事——比如你那时候肯定是Agent。产品出来前可能就融好几轮钱，然后开始pivot，或者出了一个不温不火的小产品，能赚点钱，后面新概念出来又去做新产品，不断尝试。

我觉得这可能很多人会对号入座，但确实是很典型的流程。到今天，我觉得是过去几年软件行业的一个低点。23年肯定很低，24年下半年也很低，但24年具身还是蛮热的。到25年具身非常热，AI硬件也起来了，很多人觉得具身泡沫太大开始质疑，但今年具身比去年还热。

**梦琪:**

但今年的软件不如去年热。首先软件行业特别惨，看二级市场的股价也能看出来。然后说到融钱这个事，我觉得去年从5月份之后，融到大钱的软件创始人都应该给Manus磕一个，因为是它让Agent这个事在国内有人愿意看、愿意投。

其实那一批做Agent的公司，很多都是To B Agent。To B Agent跟SaaS本质上没有区别，产品不一样，但商业模式有什么区别？最后还是舔那些老登，让大家来买你的产品。流程上、商业模式上没有太大创新，就是产品不一样，就能拿更多的钱、拿特别高的估值。

既然说到Manus，再说另一个点。 **今年二三月份的时候，我都有点扛不住了，因为不知道做什么。做什么我都觉得回答不了你跟Claude Code的区别是什么，回答不了Claude干你这个事你该怎么办，甚至回答不了Claude生态内有人做个skill想干你。没有一个站得住脚的独特差异化和** **核心竞争力** **。** 二三月份就觉得我要死了，觉得做软件的就是脑子进水了。

**曲凯:**

这里面一个核心节点就是AI coding的起来。但去年同期还没这么明显，大家不会有像今天这么强的质疑说模型就能干一切，或者几个人用一个skill两周就能做一个之前半年做的产品。

**梦琪:**

Coding变强这个事是指数级的。我年初跟OpenAI做post-training的同事聊，他说去年年底他们还不太用，到今年二月份，OpenAI内部post-training百分之八十的代码是Agent写的。但他也感慨自己并没有因此变得更轻松，因为要做的事变得更多了。

我也替我们这个行业的创业者挽个尊——确实没有人能预料到模型发展到如此强大，以至于所有软件的逻辑好像一周以内能搓出一个跟别人差不多的东西。

00:05:33

但我还是想讲一个相反的逻辑。 **这个"谁谁谁用什么多长时间就能把你这事干了"听起来特别顺理成章，甚至投资人质疑新创业者都是这么说的。但你真的仔细回想，这一年里所有做过PR的软件公司，产品能用的、用起来不挂的、在你电脑上能顺畅玩起来的，不到百分之十。**

Manus刚出来的时候大家说这想法我两年前就说过，不就是给他一个沙箱，很简单。但到今天Manus出来一年多了，谁能在工程质量和稳定性上做得比较好？这个市场挺有品牌效应的，Manus、GenSpark，可能我都不会试第三个了。

**曲凯:**

你这正好讲的是第二个问题。当下软件颓势其实就两个点：一是模型太强了，大家觉得软件不行了；二是实际上能做出来好产品的很少。但我对这个事是乐观的，没有什么时候是一家公司可以吃到这么大市场的，这里面一定有机会。不过另一个问题我确实也挺难解释——如果AI这么强，AI coding这么强，产品这么好做，为什么过去别说一年，过去三年真正出来的、大家认可的产品仍然非常少？

**梦琪:**

我自己有一个理论。 **大部分的产品是没有体验而言的，哪怕你是个wrapper都没有体验而言。** 诈骗公司特别多， **他所讲的能帮你deliver的价值和你自己用下来的价值不匹配。所以这个行业里有些产品火了非常应该，因为人家能deliver他说的那个价值，不是因为他有多么独一无二的idea。**

我有一个比较悲观又比较乐观的判断： **未来就是全明牌，所有赛道都是明牌，不存在你想得到我想不到的idea。那就是在明牌的赛道里面，每个赛道可能会有一家真的在乎体验、真的会做产品的公司出来。这是我乐观的部分——每个赛道都有机会。但悲观的部分是，真的好难有特别明显的、特别大的差异化。**

**曲凯:**

但事实上不是这样的，事实上能用的产品很少。如果是你说的前提，应该是市面上一堆产品大家都在用，那我们可以说很难差异化。但现在不是这样。所以正好以你自己公司为例，你们做了一年左右，之前也转型过好几次——这很正常，现在公司基本没有不转型的。但这个过程中到底发生了什么？你们一开始定的方向为什么没有发出来？为什么要转型？中间发过的小产品为什么没做起来？

00:08:45

**梦琪:**

我们在成立之前先做了一个决策——做vertical agent还是general agent。我们决定做vertical，原因很简单，觉得general好拥挤，那会已经有第18个Manus出现了，不想加入混战。

我们当时有几个决策做得特别坚定，觉得自己太英明神武了，逻辑上太自洽了。 **但后面发现那些特别对的决策都是错的。**

**第一个错误决策：选了sourcing agent，大白话就是找人agent。** 我们当时想做达人营销切入，逻辑是你找明白达人了，就能找明白候选人、专家，甚至能找明白客户在哪。 **我们认为sourcing是一层能泛化的、有场景迁移能力的场景。**

**曲凯:**

我打断一下，你刚才说差异化那个事情， **我觉得现在没有差异化的不是产品，是叙事。** 包括很多人跟我讲要做什么事，我都觉得已经听了好几年了，最后没有人做出来。 **比如sourcing agent，现在满大街都是，很多人在做在讲，过去几年也一直有人在讲，但没有人能做出好的结果。**

**梦琪:**

后面会讲为什么sourcing这个事绝对不成立。我们选了达人营销先开始做，做了两个月上线了， **得到一些反馈后遇到一个分叉：两类人给你反馈，一类是创始人——我们叫非专业用户，一类是公司里负责达人营销的人——我们叫专业用户。两边提的需求完全不一样。**

创始人说：别跟我说中间需要我点什么按什么，我什么也不想按，你给我结果。专业用户说：中间得可审计，这一步需要我确认才能进行，指标我要自己定义、自己筛选。

当时市场上非常火的一个概念是——你要蒸馏专家的经验和知识，做成产品后服务非专业的人。本质上是把高认知的能力浓缩成产品，再卖给非高认知的人。

又过了一两个月， **继续做这个产品，突然发现这玩意长得就是SaaS，** 就是这个行业里八年前那些SaaS公司产品的长相。 **那一刻就觉得完蛋了——你说要创业，结果自己没有高质量数据，用人家的** **API** **、人家洗过的数据，做了一个跟人家差不多的界面，纯属脑子有病，等于把人家SaaS走过的路又走了一遍。**

问题出在哪？ **第一个点是，如果选定了服务专业用户，你就是在给他做工具。** 打个比方，你给一个专业剪辑师问他需要什么样的agent， **他会一直提需求，直到你长得跟Adobe差不多才收手。因为他对过程的把控非常严谨，这必然走向SaaS。甚至我们当时还不如原来那些SaaS竞品加个AI** ——它们就是SaaS产品右边加一个弹出来的bar，在里面问AI问题。

第二个原因是 **sourcing这个事本身不成立。** 你想用AI解决sourcing的问题， **但sourcing只占用户工作的百分之三十。** 等于在巨大的价值链条上选了一个对AI来说的low hanging fruit，后面来回沟通、最浪费时间和人力的部分被skip掉了。

我们当时还有一个梦想， **希望能替代agency的工作——agency给客户交付什么，我们的agent就应该交付什么。但一旦陷入跟专业用户的沟通，你做不了那事了。** 组织就是在听专业用户提需求，你说不管这个用户的需求，听着就特别政治不正确。所以这是我们大概做到十月底遇到的困境，做了三四个月的时间。

00:13:21

于是我们开始想办法不要让产品这么 SaaS 化，要尽可能交付结果、尽可能延伸。 **比如我们当时只做 sourcing，但高价值点不在 sourcing 上，可能在下一个步骤 communication 上。** 所以我们就说得把后面几个环节都做了。做的时候又遇到一个问题——你要拓场景了，因为单做一个场景喂不饱这个公司。你是往找人 agent 方向发展，还是立足达人营销往其他增长手段去拓展？

**曲凯:**

这里我搞清楚一点，你说喂饱这个公司是从融资叙事的角度，还是从什么角度？

**梦琪:**

纯是为了多挣钱。

**曲凯:**

但如果这个市场很大，你们当时体量在市场里面是非常非常小的，为什么不能先把某个事情做到足够好？

**梦琪:**

**我们当时对搜索这个场景比较悲观，因为这个事情百分之七八十决定于你的** **API** **数据质量。**

**曲凯:**

所以其实你说拓的时候， **背后潜在含义是你已经觉得现在所选的方向不是那么对。**

**梦琪:**

**绝对不是高价值点。等于就是一个特别复杂的活，你干了它比较简单的部分。**

**曲凯:**

**那如果纯理性的话，其实应该把那个完全去掉，然后做新的东西？**

**梦琪:**

这是个非常有趣的思维实验。

**曲凯:**

因为从你的叙事讲出来是觉得那个不够喂饱所以要拓。但经过两轮问答， **我的感觉是你实际的想法是那个事就是不行，所以你是通过拓的方式来延缓了对之前事情的否定和放弃。**

**梦琪:**

站在那个时间点去看，它不是零和一的区别，而是我们觉得当时的价值点不够 strong，是强度问题，不是能否问题。所以当时决定好，我要去把高价值的点给它做好。

00:15:09

当时就两个方向。一个是纵着打，把达人营销这个事打深，往全链路去做。第二个是横着做——你会搜达人了，那你就应该会搜候选人、搜客户。

我们在这个决策上做对了。我们决定把增长这个事做深，原因很简单。这就好比我现在掌握了面粉和鸡蛋这两个原材料，然后我说我这个店铺要卖大饼加鸡蛋和意大利面——非常 make sense，因为都是面粉和鸡蛋做的。 **但你会发现没有人同时来你的店说"给我一份大饼加鸡蛋，再给我一份意大利面"，没有这样的逻辑。**

**所以这是我们没有选横着打找人 agent 的原因。找人 agent 非常难做 cross sell，谁会同时有找用户、找客户、找达人、找候选人的需求？只有 founder。但 founder 自己是不会用产品的。所以在大饼鸡蛋这个逻辑下，横着打肯定不成立，我们就决定做深——在增长这个场景上，让模型的智慧比通用模型更好一点。**

于是我们开始做 GU，开始做 Reddit 等其他增长动作。这个过程中我们也交付了几个大客户的项目。 **但做到12月份的时候，我非常清晰地感受到垂直 agent 可能不成立。这是非常难以承认的事，因为我们创业时跟所有投资人说我们要做垂直 agent，过了六个月发现这个事屁用没有。**

可能有些垂类是成立的，比如合规之类的。但我之前在一家创业公司工作过，再加上这次创业， **我们发现大部分垂类 agent 面临一个结构性困局：你的客户其实不用你的产品，但你又想挣钱，所以就变成你出一个人去服务他。垂直 agent 公司在过渡阶段都被迫成为 agency。**

我甚至发现更极端的案例： **一些做垂直 agent 产品的软件从业者，产品是给投资人做的、给他们交作业就好，真正挣钱靠服务。而且内部服务的人也不用这个产品。产品纯属一个景观效应——当成一个景，给投资人看看。**

第二个系统性困局是， **说破大天了它是个 ToB 业务。ToB 的组织形态跟我们这批年轻的创意型公司就不一样** ，加上 ToB 周期长。在中国做 ToB 是个贬义词，非常不好做。在美国做 ToB，华人的天花板非常明显。这句话说出来肯定招人骂，但这是我专门过去调研了一个多月得到的比较痛苦的结论。

**曲凯:**

这是客观现实，很正常。就像一个印度人来中国做 ToB 也很难做一样。但我想问的是，那个时间点大家讲的都是做 Pro-C，为什么你一直在提做 ToB 而不是往那个方向走？

**梦琪:**

**因为大家经常把 Pro-C 和** **OPC** **混在一起，我们也犯过这个问题。** 比如我们想，这么多人都创业做 AI 产品，都没流量都想增长，那我们做个增长 agent 不就是卖铲子嘛？这逻辑怎么听怎么对。但真做的时候，首先我们混淆的概念是趋势和现实—— **OPC 是趋势，但不是现实。现实中 OPC 首先很少，挣到钱的 OPC 更少，他没挣到钱凭什么给你钱？** 做独立开发者生意其实挺艰难，因为独立开发者首先要自己先吃饱，才能拿一点钱出来一起搞增长。

**第二是我一直觉得最有信息获取能力的 Pro-C，可能自己就能把 Grok 用特别好，** 留给我们的机会也没那么多。

00:19:49

后来我们转型时觉得可能有两个方向。第一个方向是做 token 生意——找一批 ARPU 特别高的用户，比如一个月消费5万美金的大R。我前两天还看到一个同行，只做一件事就是大R运营，让这些月消费5万美金以上的人留在自己的平台继续聊天。这种生意用户量不需要太多，我年底调研 Lululemon 的时候发现这个公司用户量其实没有想象中多。

**曲凯:**

现在很多 AI 公司就是这样，ARPU 很高，客单高。

**梦琪:**

对， **一两万 power user 就完全够吃了。** 第二类我觉得就是健身房——他给你订阅费，你赌他不来。或者赌他 token 别打满。因为在高竞争的情况下你很难留特别高的毛利，这种业务很难有超额收益。

**曲凯:**

这种典型的产品是什么？

**梦琪:**

我们其实在一个很奇怪的渠道上看到过——Steam。我们做 C 端转型的时候，希望产品更有人格、能跟用户有情感连接，提供情绪价值，因为做效率工具大家都是一样的烂。在 C 端产品里，离情绪价值最近的是游戏，我就批量跟在美国做游戏的同学学习请教。

**我就问他们：游戏是一个赛道，生产力是一个赛道，中间有一堵很高的墙。** 我们这个公司如果想打破生产力和情绪价值的界限，有没有这种探索的可能性？ **那个哥们说"大姐你真是孤陋寡闻，其实早就没有这个界限了"。Steam 上有很多这样的产品，我们去试了一些，确实挺打开眼界的。**

**曲凯:**

你能简单讲一两个吗？

**梦琪:**

**最典型的几个效率工具——番茄钟、记 To-Do 的清单类产品、个人习惯培养、给 ADHD 的产品，其实都有。它们用游戏化的方法来做。** 第一印象它是个陪伴，每天 default 会打开它，甚至有人卖动态桌面。你很难定义它是个游戏还是生产力产品，品类会被模糊化。尤其在软件如此容易被创造的情况下，你的上限就是想象力。

00:22:48

说回 ToB 那个事，大概在二月份我们就低功耗经营了，只留了一点维护的人力。决定不做有两个点： **第一，如果做 ToB 我就不应该创业、不应该融资。** 我以前做过几年 ToB，看看之前的客户资源加上在 AI 行业认识的朋友，其实不需要融资。 **第二，我觉得这不值得创业。** 创业应该追求一个比较高赔率的事，这个事没那么让我足够 excited。

它更像是我站在半空中，下面有个棋盘，棋盘有各种格子。你在格子里看到这有一机会、那有一机会，盘了一遍发现这个机会比较适合我。它完全是一个逻辑追问、机会追问的事。它不是"老娘今儿就干这事了，老娘今天不干这事我得难受死我得后悔死，你给不给我钱你爱给给你不爱给滚蛋，我就要干这个事"。我当时觉得我不能再以这样的状态创业了，不能把它变成一个证明题。

这个产品我能说出一百个替代品的名字，太多了。但就这么简单一个产品，仍然有非常多用户喜欢。有个南非用户说，用了这个产品他终于能沉浸式干一件事了—— **他写文档时能直接at某个tab把信息拿过来，甚至at Gmail把客户的上下文贴过来，不需要再去找** **碎片化** **信息来充斥大脑。** 我自己也是ADHD，特别珍惜专注的时间，特别讨厌每天在屏幕上划来划去，让焦虑一点点叠加。

过完年我们做了一些改版，风格我非常喜欢。后来看到Flot，发现大家审美真的很趋同。但Clico的局限性是只能在浏览器里运作。我整个电脑不只是开浏览器，还开了Lark、Slack和各种窗口，它不能在整个桌面上运行。所以我们顺理成章地说，应该扩大它的行动范围，做一个客户端产品。

那会我经常跟大家说，大家都说谁谁谁是套壳的，我觉得应用公司都是套壳的，但就怕有些公司套一些比较老的东西。所以我们当时坚定套Claude，千万不要套上一代的东西。我们套了Claude做了客户端产品。

大家都说做一个产品可以特别快，一周就能出。我现在对这种言论嗤之以鼻。就这么一个客户端产品，听起来蛮简单的，我们做了三个月，截止到昨天晚上上线了第49版。到现在还有一些bug，但稳定性我认为是目前我看到的客户端产品里非常高的，能稳定输出、能用。

越做越觉得，在coding agent越来越强的情况下，反而对软件更充满敬畏之心。它就是由一万五千个细节组成的，你就一点点把那一万五千个屎吃掉，多吃一个用户体验好一点。

**做完这个产品我反而不焦虑了。** 二三月份我焦虑裂开，觉得真的脑子进水了要干软件，那会同情所有做软件的人，经常抱在一起聊，感觉下一秒就要抱在一起哭。 **但等做了这个产品，每天看着待修的问题列表，就觉得把这些屎吃掉就好了，吃得快产品体验就更好。**

我们3月底4月份就公测了，那会即使公测，它也是不能用的状态——能打开能运行，但八百个问题，用户倒了血霉了赶上我们这样的产品。

Clico做的时候我一开始挺看不起这个产品，觉得不就是一个插件，功能非常简单，没什么意义。但后来用户反馈加上我自己学C端产品逻辑， **我发现只要你能缩短用户意图和解决方案之间的距离，你就有价值** 。

为什么Clico有价值？原来用户想用AI写东西，要先截图发给ChatGPT，写一个prompt，等返回结果再复制回来。听起来好像也有解决方案， **但Clico就一个按钮，你按一下不用说任何话，这个按钮就代表他的意图，不用写prompt，直接给结果。** 你看虽然大模型已经如此强了，但好多人还是会有个翻译插件，就是解决这么一个意图。

当你想解决的意图变多时，怎么编排、怎么保持屏效健康、让用户接受信息的复杂度不要过高，这是产品体验上需要权衡的。我现在对C端粗浅的理解就是——你为什么打优享不坐拼车？ **C端产品最后就两个字：体验。只要你能保证体验比直接用** **ChatGPT** **好、比直接用Claude好，你就有活着的价值。**

**所以我们花了非常多时间做用研，主要阵地是Reddit。** 一方面在Reddit做数据挖掘，但这只能给你静态结论，不能给你跟着思考路径一起变化的动态答案。所以我们在Reddit发了大量帖子——每一个产品决策都拆成决策漏斗，每一层都有对应要发的帖子，每个问题大概40到50个帖子发出去，再回收看反馈。有些帖子跟产品相关，有些不提产品单纯讨论需求和痛点，因为提了产品流量不好。 **做了大量验证之后我们才去做能力。**

**我推荐所有创业者都去泡Reddit，因为一泡你就不焦虑了——你会发现很多问题没有被解决。** 比如context switching的问题，每个人都在吐槽说每天在几个context里切换，脑子要分裂了。这些问题每天都在被提起。用Reddit做用研阵地和验证阵地，非常解决我们做优先级的问题——这需求是不是真需求、是不是刚需、有没有人会付费、付多少钱、以前产品哪里不好。这些给了我们特别大的助力，让产品做得更扎实、更克制。

**梦琪:**

我觉得这是共性。这个行业里决定软件创业者精神状态的最大牵引力就是AI模型的发展。这个最大牵引力发生变化时，所有人都会同等权重地受到影响。大部分人都被迫一直在pivot，做着新产品又在想是不是做着做着又不用做了。

我们经常开玩笑说，我们做一个产品，要适配不同的版本和型号，团队但凡没点定力和没融到比较多的钱，都不太能有信心坚持到最后。 **我们做这个产品的时候在想，打磨到不错可能需要3到6个月，但产品寿命会不会只有一年多？那剩下的6个多月是能挣钱的，得赶紧想办法挣钱。它完全改变了我们的思维方式和** **商业模式** **——你不再像以前一样** **画** **特别长时间的规划，别提明年了，就下个月吧。**

这个产品我被气哭过好几次，太难做了，非常多的细节要调。有一次在Palo Alto的Blue Bottle调产品，没调出来，我气得捶了两下桌子，隔壁桌被震动，两个人站起来说"你为什么这么生气？能让你这么emotional的一定是个好玩的东西。"然后我就把产品介绍给他们了。所以我们这一批人都不得不受到模型的影响，但这可能也是创业的乐趣吧——变化的因子太多了。

**曲凯:**

但你这么讲，投资人听到更不敢投了。现在现状也还是这样的？

**梦琪:**

我觉得这个行业现状就是这样。

**曲凯:**

有解法吗？解法就不要做软件创业，去做具身。

**梦琪:**

**可以啊，就卖数据。**

00:50:20

**曲凯:**

我觉得不管是你，还是Slack还是其他现在比较有水花的产品，跟之前一个很大的区别是：你们在解决AI时代自身内部的问题。你不是在说企业怎么增长——这是一直以来都有的问题，SaaS在解决，AI进来可能不一定解决得更好。但你现在解决的是，比如大家已经在用AI了，怎么跳转窗口的问题；Slack和很多产品解决的是本来就在用AI coding，怎么用得更好的问题——这是新场景、新需求。

**梦琪:**

第一个我不同意。我们现在的很多用户反而不是非常典型的early adopters。我们已经做出了选择——想做健身房，不想做一个特别没有毛利的业务。

**曲凯:**

但你现在的用户，或者未来瞄准的用户，是不是本来就在用AI了，你只是帮他们用得更好？

**梦琪:**

其实很多用户没有。 **我二月份做了一些田野调查，在旧金山所有活动上逢人就问：你知道Vision** **Pro** **吗？你知道豆包吗？** Vision Pro是在101公路上打了非常大广告牌的产品——只有40%的人知道，这40%里面可能只有20%在用、已经付费。还有另一个创业者做的田野调查是路边随机问人用不用ChatGPT。

**我非常乐观的一点是渗透率不够高。** 你不一定非得说你是AI产品，你就说我要解决你的问题——今天这么多问题放在那没有人去解决。

我悲观的部分是一些特别简单的小效率工具的前途，包括Clico。假设全人类都有创造的欲望，这类产品的机会完全不存在。 **但首先不是全人类都有创造欲望；第二不是所有人都能access到Claude Code去做这个事，** 很多人听到code这个词就不想用了； **第三即使做出来了，它不是做完就完了——像是草坪，经常要修剪。** Bret Taylor说的， **修剪的成本比把草坪放在那的成本还高。** 我自己vibe code的小工具，维护起来总遇到corner case，创造的快乐结束后都是痛苦。

不过以后维护是不是也能自己维护了，corner case自己发现自己解？这也是有可能的，也是我们最近在试的方向。我们一二月尝试了一个多Agent互相协作的产品，最后没发。原因特别简单： **我不想继续发一个平台类型的产品等着用户来演绎，我知道自己不擅长做社区。**

但最近，比如用户提一个bug或问题，内部这些问题的流转我们是用这套架构做的。我更希望它先把内部这一整套做明白做清楚，有一天别的公司想用能给他。现在AI的软件开发流程就三步：发现问题、解决问题、验收。越来越没有人关注解决这个过程，只有两端需要倾注注意力——发起一个问题和验收一个问题。

我们现在研发主要卡在验收：简单的UI问题AI自己能验收，复杂的产品问题可能还得人来验收甚至用户来验收。发起这一端，是用户的context、用研的context、Reddit来的context、产品经理的想法。然后由Multi Agent决定什么事先做什么事后做，做整体规划，bug的P0、P1怎么设，分发到能解决问题的人身上——其实这些"人"都是研发同学自己的coding agent。所以我们现在是人在两端，甚至发起端也不需要人。

比如用户提一个需求或bug，Agent判断，分配谁来修，修完后Agent自动给那个人发邮件。我觉得这某种程度上是新时代的Jira。但我没想清楚的是它应该是什么样的产品界面——我有一个清晰的答案：它一定不是个chat。那如果不是chat它应该是什么？在没想清楚之前我不会发这个产品。我要确保它能把内部问题解决得非常好再去发。我对做平台类型的产品不太感兴趣。

00:55:38

**曲凯:**

我刚才在听你讲自己的整个过程，很多都很有代表性，但有一件事你好像没有做——去年那一波大家普遍会做一个很大的宣发，说"这是全球首款XX"，这个事你没有做？

**梦琪:**

完全没有做。一是去年因为一些原因，我还是得低调一点。第二是我看到除了Manus以外，做世界首发能拿到比较好的流量和留存的产品，其实很多是一波流。我觉得有点浪费钱，也有点太强的交作业意味——你交完作业溜了，用户留存不好，一波就过去了。我比较抠。

但最深层的原因，夜深人静的时候我认真问自己，就是因为我对自己产品不满意，或者说我不相信。所以现在这个模式变了，这个产品再垃圾，不管你怎么骂我，我就爱这个产品，这就是我的Spark。人内心深处的那种火、那种光，那种Spark，AI代替不了，这是人身上最宝贵的部分。

所以这个产品甭管它多丑，甭管现在还有些问题，我会倾尽全力去做它。因为我很爱它，我自己用得很爽，用户也用得很爽，它不仅有解决问题的价值，也有情绪的价值。现在不够好，我们改嘛，你们提我改，我是个听劝的人。但这个火苗是我没有办法拒绝的。

上一个产品，我不得不承认那是一个机会主义者，一个迎合市场的人，想交作业、想做证明题的人。今天这个产品，我想做个简答题。包括我们后面分享的自己做增长的那个skill，大家都觉得很好用——这些过程产品是我内部用很爽、已经验证的东西，我很有信心也很喜欢。这些过程产品能代表我们公司的精神气质，也第一次让我有机会把个人的世界观、价值观、性格投射到一个产品里面。

**曲凯:**

我觉得这就很好，你逐渐找到了自己的calling。

00:58:27

**梦琪:**

**有的时候噪音确实挺多的，非常多的分析。我那一年听了太多关于** **大模型** **的预测，跟researchers私聊，大家都有比较多的判断，但其实很多声音真的是噪音。**

**曲凯:**

我今年有个特别明显的转折点，过去三年一直在follow各种进展， **然后到Harness出来以后我真的麻了，不想跟了。** 概念太多了。

**梦琪:**

创业者特别容易被信息轰炸，这市场消息真的太多了。 **Harness这个事我说实话也有点麻，但不得不去学、去看。** 包括我们今天这个播客，我希望不要给这个行业增加更多的slop。我希望要么是能让大家没那么焦虑。

软件虽然这么强， **但软件真的是一块需要持续维护的草坪** ，不要太悲观地觉得三秒钟它能做一个跟我一样的东西干死我。 **第二，这个世界非常大、非常参差。** 比如我们当时做增长Agent的时候，非常坚定地选Agent而不是workflow—— **你不得领先吗？作为AI创业者怎么能选workflow呢？肯定有技术歧视嘛。但你真的仔细去看，对于大部分企业来说，能把workflow用上就已经能解决大部分问题了。** 有的人AI到现在还拿它当简单助手呢，需求是多分层的。

我特别讨厌一类文章，写那种耸人听闻的标题，我觉得写这种文章人脑子有病。商业社会里所有竞争都不是一瞬间完成的，不是抖音出来快手今天立刻死，不是某个产品出来做workflow的公司立刻死。这个世界是在动态博弈、动态竞争里运转的。 **现在自媒体特别喜欢把一个漫长的、可能有点枯燥的动态演化过程压缩成一幕几秒钟的短剧剧情——我愿意称之为AI行业的番茄、AI行业的红果。它想制造非常强烈、非常快速的戏剧冲突。**

**但真实的商业社会就是有大量不需要最领先技术的公司，美国、欧洲很多公司数字化都还没上完呢。** 这个世界是很动态、很参差、很分层的，不是一个只有early adopter活着的地方。所以我反而挺充满期待的—— **只要行业渗透率还足够低，** 虽然行业结构决定了大部分利润会被御三家吃走，但剩下的公司喝个汤其实也还不错。 **甚至因为这个动量太明确、渗透率太低，你喝的汤可能比过去做SaaS吃的肉还要多。** 所以我还是比较乐观，所有特别焦虑的人欢迎来找我聊天，我能把你搞得不焦虑。

01:01:38

**曲凯:**

你之前从字节出来，刚才也提到吃肉跟喝汤的问题。最后一个问题，如果现在回到在字节的时候，大概一年前，让你重新选、重新过这一年，你会怎么做？会不会继续留在字节？

**梦琪:**

这个问题太deep了。其实在我最痛苦那两个月，无数次问自己你为什么创业，不创业是不是还挺爽？我当时那个工作确实挺爽，也是比较擅长的事。官方版本是我还是会创业，我很enjoy创业、很擅长创业，meant to be就应该干这个事。 **非官方版本是，说实话三四月份的时候我不想干了，觉得做这个破软件真的没有意义** ——那时候产品刚开始做，遇到特别多技术上的困难，To B也不做了，看着这个行业陷入了无尽的虚无，那是我最痛苦的时候，真想把桌子掀了算了。但最近热情满满，我觉得我要是做不出来别人也做不出来，但凡这行业有一个人做出来，那一定是我。

**曲凯:**

这挺正常的，心理上的up and down嘛，每个人都是这样，但底色还是你就是一个热爱创业的人。然后关于融不融资的问题，我经历过十几年前的创投市场——天使轮是1000万人民币估值投100万，当时默认的是先用100万把产品做上线，有了数据验证再融下一轮。跟现在挺不一样的，当时产品上线用户不用花钱，免费获得用户看DAU， **现在是看** **ARR** **。**

**梦琪:**

所以我们觉得足够幸运，也足够臭不要脸，在这样的周期里面拿到了钱。有时候我拉长时间看这件事，好多人觉得你得持续去融，因为融资也是比赛，做产品也是比赛，你所有的事都是在竞争里面。但我一直想找创业里面真正的充分必要条件。

**融资好不是充分必要条件——新消费那一波，融得最好的可能现在最惨，回购回购不掉，收入扛不住估值，投资人和创业者都很痛苦。团队有大牛也不是重要条件——有的挂个名，有的落地不够好，有的落地好了赛道不好。** 我后来越来越发现创业这个事挺玄学的，手上有什么牌把什么牌打好，很难特别科学地看待。

我之前去过一个非常早期的创业公司待过一年，也是明星创业者。我最大的感受就是，他融了这么多钱，也不能说因此就比融钱少的人发展得更好。所以对创业这件事，原来我希望有一个科学的方法，能找到一些因子、一些变量。但我现在觉得找不到，用心去做这个事情，而不是用脑子，左脑不能用太多，然后把你的用户当人。

01:05:15

我特别不想做效率工具。效率工具关注的是你的问题，它不管你曲凯是谁、你是什么样的社会关系、你今天心情怎么样。我想做的产品是，首先我能解决你的问题——不解决问题就是一个只提供情绪价值但不靠谱的下属，没有意义。同时我在乎你是谁，在乎你现在的心情，把你当个人看。

DAU对我来说不是一个在变化的四位数、八位数。背后都是人，跟每个人聊得足够多，我就能知道你的家庭，你在心里会对这个人有更立体的形象，你会更加体恤这个人，跟这个人的connections也会更多。我觉得这是人和人之间、AI和人之间最珍贵的部分。

**曲凯:**

**我们今年播客重采了一些之前上过的人，包括文峰，Albert也今年来了两次，大家一个很明显的变化就是都变得更务实了。之前可能比较理想主义，基于理想主义去做一些务虚的东西、讲一些概念。现在都不搞抽象了，我觉得这很好。** 但同时一个好的创业者是务实的同时能保持理想主义。

**梦琪:**

对， **我现在很讨厌搞抽象，我觉得这就是装逼为主，对解决问题没有帮助。造词的、讲很虚幻东西的我都很讨厌。** 所以好多人问你这产品是什么，我说就是Claude套壳，然后提供点情绪价值。

**曲凯:**

**造词加理想主义是投资人的风格，这没错。但如果创业者变得跟投资人一模一样那就完蛋了。** 好的创业者是能把投资人的语言、投资人的逻辑、自己的理想主义、还有务实落地都结合好。

**梦琪:**

是，所以我非常乐观，软件行业还大有可为。
