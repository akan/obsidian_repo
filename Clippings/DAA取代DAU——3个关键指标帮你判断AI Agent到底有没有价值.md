## DAA取代DAU？3个关键指标帮你判断AI Agent到底有没有价值

> **核心答案** ：是的，DAA（日活智能体数）正在取代DAU成为AI时代的新度量衡。但不能只看一个数字——你需要用"DAA规模 + 任务完成率 + 单任务价值"三维模型来判断你的AI投入到底值不值。

上个月，一个做数据分析的朋友跟我吐槽：公司去年上了AI工具，管理层说"总算力成本涨了5倍"，但问"到底创造了多少价值"，没人能说清楚。

这其实不是他一个人的困惑。我最近跟几个企业数据负责人聊，几乎个个都在算这笔账——钱花出去了，效果在哪？

2026年5月13日，百度李彦宏在Create大会上抛出一个新概念—— **DAA（Daily Active Agents，日活智能体数）** ，说这是AI时代的"新度量衡"。消息一出，科技圈炸了锅。

有人说百度又在造词，有人觉得确实戳中了痛点。

我用了一周时间，把这个概念从头到尾扒了一遍。结论是： **DAA不是在取代DAU，而是在回答一个更根本的问题——你的AI到底在创造价值，还是在消耗预算？**

这篇文章给你三个抓手，帮你判断。

---

## 一、为什么DAU和Token都不够用了？

先看一组让我印象很深的数据。

**Anthropic（Claude的母公司）全产品DAU总和，只有OpenAI（ChatGPT）的2%。** 但它的年化营收（ARR）却反超了OpenAI。

DAU高的反而赚得少？这说明在AI时代，用户规模和商业价值之间出现了断层。

再看Token。Token衡量的是"消耗了多少算力"，不是"创造了多少价值"。你想想，用户和AI闲聊半天，Token烧了一大堆，但一个实际问题都没解决。消耗高，产出低，这就是错配。

Uber今年就栽了跟头——6000名工程师，95%在用AI编程工具，70%的代码由AI生成。看起来很酷对吧？结果全年2400万美元AI预算，4个月就烧光了。

CB Insights的调研更扎心： **80%的高管说推进AI Agent是优先事项，但40%的人根本不知道ROI怎么算。**

所以问题来了：到底用什么来量AI的价值？

---

## 二、DAA的三维价值模型：从"数人头"到"数活"

李彦宏给的定义很直接： **DAA衡量的是每天有多少智能体在给人类干活并交付结果。**

不是数多少人用了AI，而是数多少Agent在创造价值。

但这只是一个数字。真正靠谱的是背后的 **三维模型** ：

### 第一维：DAA规模——有多少Agent在干活？

这就像移动互联网时代的DAU，但数的是智能体不是用户。

比如一家银行的智能客服系统，一天处理了12万个用户请求。每个请求背后是一个Agent在运行。那它的DAA就是12万。

但这只是基础。更大的数字是李彦宏的预测—— **未来全球DAA可能超过100亿** 。今天全球DAU最高的Meta是34亿用户。100亿智能体意味着什么？意味着每个用户背后可能配着3个Agent在替TA干活。

### 第二维：任务完成率——干成了多少活？

数出12万个Agent在跑没用，关键是有多少真的把事干完了。

百度开发者中心的一个案例很有说服力： **某物流调度系统引入了Agent有效性判定机制后，无效任务识别准确率从60%左右提升到92%，资源利用率优化了37%。**

判断Agent有没有干成活的逻辑，其实不复杂：

```
有效任务判定条件：
1. 输入格式正确（Agent理解了需求）
2. 输出了可接受的结果（活干完了）
3. 人工确认为正向（人类觉得ok）
```

三条都满足，才算一个有效DAA。

### 第三维：单任务价值——每件活值多少钱？

这是最有意思的一个维度。

举两个极端例子：

- • 某智能客服：DAA 12万/日，但每单解决的是"查余额""改密码"这类低价值任务
- • 某法律咨询AI平台：DAA才1.2万/日，但ARR达到了2.8亿元

**法律平台单位DAA的价值，是普通客服的23倍。**

这说明什么？DAA不是越大越好。 **高价值任务 × 高完成率 × 适当规模 = 最优解。**

百度开发者给了一个任务权重参考：

| 任务类型 | 基础权重 | 复杂度系数 |
| --- | --- | --- |
| 文本生成/简单问答 | 1.0 | 1.2 |
| 图像识别/分析 | 1.5 | 1.8 |
| 决策优化/数据分析 | 2.0 | 2.5 |

决策优化类任务的基础权重是文本生成的2倍，因为它的单次价值更高。

## 三、这套框架怎么用在数据分析场景？

说到这，你可能觉得DAA是给百度这种大平台用的。其实不然。

拿我们自己的FocusGPT来说——它是一个数据分析智能体，你问它"上季度哪个产品线增长最快"，它自动查库、分析、出图、给结论。

如果用DAA三维模型来评估它：

**DAA规模** ：FocusGPT每天被调用了多少次？不是打开页面的次数，而是实际发起了数据分析任务的次数。

**任务完成率** ：这些请求里，多少是真正完成了分析并输出了可用结果的？用户追问了3轮才出正确结果，算完成还是算部分完成？

**单任务价值** ：一次数据分析请求，帮用户省了多长时间？传统方式自己做报表要1小时，FocusGPT 3分钟出结果，单次节省57分钟。按分析师时薪算，每次调用创造了约XX元的效率价值。

这个视角让管理层能说清楚一件事： **每个月花在AI工具上的钱，换回了多少"有效分析任务"和"节省的人力时间"。**

DAA不是百度的专属概念，它是每个用AI的企业都该建立的价值评估思维。

——写到这我突然想到，这个框架其实也适合用来评估我们自己。回头用DAA给FocusGPT算一算，看看到底值不值（笑）。

## 四、DAA的坑和边界

说清楚DAA的价值，也得说清楚它的问题。

**问题一：DAA很容易被注水。**

如果企业把"AI生成了但没人用的内容"也算DAA，那数字可以做得很好看，但毫无意义。关键在于 **"交付结果"这个定语** ——Agent必须真正完成了任务并产生了可用产出。

**问题二：不同任务的DAA不能直接比。**

一个"查天气"的Agent日活100万，一个"做财务报表"的Agent日活1万。但后者1万次调用产生的商业价值可能远超前者。所以DAA必须结合"单任务价值"一起看。

**问题三：定义"有效任务"需要机制，不是拍脑袋。**

百度开发者中心给了一套判定流程：输入校验 → 输出质量评估 → 人工验证。从技术角度，这个可以用伪代码表达：

```
def is_effective(task):
    if not validate_input(task.input):
        return False
    if assess_output(task.output) < QUALITY_THRESHOLD:
        return False
    if collect_feedback(task) != POSITIVE:
        return False
    return True
```

逻辑不复杂，但需要在AI系统设计时就埋进去。

## 五、对AI行业意味着什么？三个趋势预判

**趋势一：AI估值逻辑将重写。**

以前看AI公司值不值钱，看用户数、看API调用量。以后会看DAA × 单任务价值。这个组合指标比DAU更接近商业本质。

百度港股过去一年涨了70.82%，核心AI业务收入突破400亿，AI收入占比逼近50%。市场正在把百度的估值锚点从"搜索公司"切换到"AI基础设施+智能体平台"。

**趋势二：企业AI采购从"Token消耗"转向"任务交付"。**

以前IT采购AI工具：A厂商每月500万Token，B厂商800万Token，买B。以后会问：你的Agent每周完成多少个有效任务？每个任务价值多少？

这是从"成本思维"到"价值思维"的根本转变。

**趋势三：DAA优化会成为新赛道。**

到2028年，73%的AI平台预计会采用DAA类指标。专门做DAA监测、优化、审计的服务商将出现。这就像2015年的数据中台——先有概念，后有产业。

## 常见问题

**Q：DAA是要完全取代DAU吗？**  
A：不是。DAU衡量用户规模，DAA衡量智能体产出。两者是互补关系。就像GDP不能取代人口统计一样。

**Q：小公司能用DAA吗？**  
A：能。DAA框架可以简化到三行：数一下你的AI每天完成了多少有效任务（DAA规模），成功率多少（完成率），每个任务值多少钱（单任务价值）。没那么复杂。

**Q：DAA会不会被刷量？**  
A：会，所以需要有效的判定机制。核心是"交付结果"——AI只是生成了内容，但没人用它，不算DAA。就像代码提交了但没上线，不算有产出。

**Q：FocusGPT的DAA大概是多少？适合用这个框架吗？**  
A：完全适合。FocusGPT作为数据分析智能体，天然可以用DAA三维模型评估。每次数据分析请求 → DAA+1，完成并输出可用图表 → 完成率计入，节省分析师的XX分钟 → 单任务价值。管理层看着这个三角模型，比看Token消耗量清楚100倍。
