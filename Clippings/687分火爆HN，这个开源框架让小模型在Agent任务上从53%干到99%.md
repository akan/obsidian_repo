## Forge是什么？

Forge是一个Python框架，核心就一句话： **确保AI Agent正确地调用工具，而不是瞎猜。**

它不替代你的LLM，不替代你的Agent框架。它就坐在中间，像一个安检员——每个工具调用进来，它检查一遍，有问题的拦住重试，没问题的放行。

它的作者Antoine Zambelli做了一个测试：同样的Agent任务，没有Forge的8B小模型得分只有个位数，加上Forge直接干到84%。连Sonnet 4.6这样的顶级模型，也从85%被拉到了98%。

## 四个关键能力

我一共跑了5个测试，覆盖了Forge的四个核心能力。直接看结果：

```
TEST 1: AI说了句话，但没调用工具   → 返回retry，告诉AI"你需要调用工具"   → 而不是直接放行，让下游报错                                                 TEST 2: AI正确调用search工具   → 放行，记录步骤   → ✅ 正常工作            TEST 3: AI跳过lookup，直接调用answer   → 被拦住！返回step_blocked   → "必须先完成lookup才能answer"   → 强制工具调用顺序                            TEST 4: AI调用了一个不存在的工具delete_all   → 返回tool_error，拒绝执行   → 防止AI胡来                                                             TEST 5: 完整流程search→lookup→answer   → ✅ 全部通过，workflow完成
```

这套机制叫什么？ **Guardrail（护栏）** 。不是你写完代码后的测试，而是 **在AI做出工具调用请求的那一瞬间，就检查它有没有出错** 。

## 三种使用方式

Forge给三种人准备了不同的接入方式：

**方式一：Proxy模式（最省事，推荐）**

一行命令启动一个代理服务器：

```
python -m forge.proxy --backend-url http://localhost:8080 --port 8081
```

然后把你的Claude Code、Continue、aider指向这个代理，它就透明地夹在中间做检查。客户端（比如Claude Code）完全不知道自己背后多了一层安全检查。

关键的是，这个代理同时支持OpenAI和Anthropic协议，Claude Code可以直接用。

**方式二：WorkflowRunner（自己写Python）**

定义好工具，选个后端，让Forge帮你管理完整的Agent循环——系统提示词、工具执行、上下文压缩、护栏检查全部自动搞定。

**方式三：中间件模式（已有循环）**

如果你已经有自己的Agent框架，不用换。导入Forge的Guardrails类，在循环里加两行代码就行：

```
result = guardrails.check(response)    # 调用前检查                    done   = guardrails.record(["tool"])   # 调用后记录
```

## 为什么这很重要

很多人在用AI Agent的时候，遇到工具调用出错的第一反应是"换更大的模型"。但Forge团队的实测数据告诉我们： **问题不只在模型大小，更在有没有错误恢复机制。**

8B小模型不加护栏，得分个位数；加上护栏后84%。这个差距跟模型无关，跟工程有关。

更关键的是，Forge的proxy模式让你现在就能用——不用改Claude Code的配置，不用重写你的Agent代码。一条命令，就能给你的所有AI工具调用加上一层保险。

## 适合谁用

- 自己在搭Agent框架的开发者——中间件一行集成
- 用小模型跑本地Agent的——8B模型+Forge，效果接近大模型
